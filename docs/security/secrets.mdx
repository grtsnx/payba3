# Secrets

Payment provider credentials can create charges, transfers, accounts, verification sessions, and access tokens. Treat them as production secrets.

## Storage

Use one of these:

- deployment platform secret manager
- cloud secret manager
- encrypted CI/CD secrets
- local `.env` file ignored by git for development

Do not put secrets in:

- source code
- examples
- screenshots
- generated docs
- client-side environment variables
- agent prompts
- issue reports
- build logs

## Naming

Keep sandbox and live credentials separate.

```bash
PAYSTACK_SECRET_KEY=
PAYSTACK_SECRET_KEY_LIVE=
SAFEHAVEN_CLIENT_ID=
SAFEHAVEN_CLIENT_ASSERTION=
SAFEHAVEN_LIVE_CLIENT_ID=
SAFEHAVEN_LIVE_CLIENT_ASSERTION=
```

## Rotation

Rotate secrets when:

- a contributor with secret access leaves
- a log or screenshot may have leaked a key
- a deployment environment is compromised
- moving from sandbox testing to production launch
- provider dashboards show unexpected activity

## Agent use

Agents should generate code that references environment variables. They should not ask users to paste secrets into source files or prompts.

```ts
createPayba3({
  safehaven: {
    clientId: process.env.SAFEHAVEN_CLIENT_ID,
    clientAssertion: process.env.SAFEHAVEN_CLIENT_ASSERTION,
  },
});
```
