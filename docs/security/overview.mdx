# Security overview

payba3 helps call payment providers. Your application still owns authorization, persistence, audit logs, idempotency, access control, and business rules.

## Main risks

| Risk                        | What to do                                                                           |
| --------------------------- | ------------------------------------------------------------------------------------ |
| Secret leakage              | Keep provider credentials in a secret manager or server-side environment variables.  |
| Unverified payment state    | Verify provider status before fulfilling orders or crediting balances.               |
| Replay attacks              | Store webhook event ids and transaction references idempotently.                     |
| Unauthorized money movement | Require business authorization for transfers, refunds, debits, and account creation. |
| Agent overreach             | Expose narrow tools to agents instead of raw provider clients.                       |
| Sensitive identity data     | Minimize logs and retention for BVN, NIN, CAC, and bank account data.                |

## Minimum production checklist

- Use live credentials only in production environments.
- Keep sandbox and live credentials separate.
- Restrict payment actions with application authorization.
- Verify webhooks with the raw request body where required.
- Verify transaction status server-side.
- Add idempotency keys or unique references to state-changing operations.
- Redact tokens, assertions, secret keys, account numbers, and identity identifiers from logs.
- Monitor failed payment, refund, transfer, and webhook attempts.

## What payba3 does

- Keeps provider code behind named channels.
- Reuses and refreshes provider access tokens where supported.
- Provides webhook verification helpers where implemented.
- Keeps provider types in dedicated type files.
- Keeps signing and private helpers out of service methods.

## What your app must do

- Decide who may initiate each action.
- Store orders, invoices, accounts, and webhook events.
- Reconcile provider state with local state.
- Handle customer-facing payment status.
- Review and approve sensitive live operations.
