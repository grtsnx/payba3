# Testing

payba3 tests provider behavior without calling real provider APIs by mocking fetch and verifying request shape, headers, signing, token refresh, and response handling.

## Local checks

```bash
bun run lint
bun run test
bun run test:providers
bun run test:e2e
bun run build
bun run pack:dry
bun run test:package
bun run test:package:bun
```

## Provider tests

Provider tests live under `test/unit/lib`.

```text
test/unit/lib/<provider>/
├── <provider>.service.spec.ts
└── config/<provider>.helper.spec.ts
```

Use provider tests to cover:

- credential resolution
- sandbox/live base URL selection
- access token reuse and refresh
- request signing
- webhook verification
- request payload shape
- response parsing
- error normalization

## Package smoke tests

Package smoke tests install the packed package into a temporary project and import it like a real consumer.

```bash
bun run test:package
bun run test:package:bun
```

These checks protect package exports such as:

```text
@grtsnx/payba3
@grtsnx/payba3/llms.txt
@grtsnx/payba3/agents.json
@grtsnx/payba3/llms/paystack.txt
```

## Docs tests

The Mintlify docs map is tested under `test/unit/docs`. The test verifies that every navigation page exists and every docs page is reachable from `docs.json`.

## Mintlify CLI checks

Run these with a Mintlify-supported LTS Node runtime:

```bash
bun run docs:validate
bun run docs:broken-links
```

Node 26 is not supported by the Mintlify CLI at the moment.
