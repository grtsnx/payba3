# CI

The CI workflow is organized as a visible map of audit, test, build, and smoke stages.

## Stages

| Stage      | Purpose                                                               |
| ---------- | --------------------------------------------------------------------- |
| Audit      | Check dependency vulnerabilities with the configured package manager. |
| Lint       | Validate TypeScript style and unsafe patterns.                        |
| Unit tests | Run provider, helper, package, and docs unit tests.                   |
| E2E tests  | Run the local application smoke route tests.                          |
| Build      | Compile the TypeScript package.                                       |
| Pack       | Confirm the npm package contents.                                     |
| Smoke      | Install and import the packed package with npm and Bun.               |
| Summary    | Publish the CI map and result overview in the workflow summary.       |

## Why smoke tests matter

Build success does not guarantee a package can be installed by users. Smoke tests catch missing files, broken exports, missing declarations, and runtime import errors.

## Required local command

Before release, run:

```bash
bun run lint && bun run test && bun run build && bun run pack:dry
```

For a full release rehearsal:

```bash
bun run test:e2e
bun run test:package
bun run test:package:bun
bun audit
```
